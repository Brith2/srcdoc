Utils::AppLauncher - Abstraction layer for launching applications

The AppLauncher class provides a cross-platform way to launch external applications.

See also: PerlLauncher

Example

use Utils::AppLauncher;

my $launcher = new AppLauncher('gedit', '/dev/null');
if (!$launcher->launch(0)) {
    die "Cannot launch application.\n" .
        "Error message: " . $launcher->getError() . "\n" .
        "Error code: " . $launcher->getErrorCode() . "\n";
}

while (1) {
    if ($launcher->check()) {
        print "App is still running.\n";
    } else {
        print "App has exited.\n";
        print "Its exit code was: " . $launcher->getExitCode() . "\n";
        last;
    }
    sleep 5;
}

$AppLauncher->check

boolean $AppLauncher->check()
Requires:
$self->isLaunched() && !$self->isDetached()

Check whether the launched application is still running.

If the application has exited (that is, result is false), then you can use $AppLauncher->getExitCode() to retrieve the application's exit code.

You should periodically call this function. On Unix, not calling this function can lead to zombie processes.

$AppLauncher->launch

boolean $AppLauncher->launch(boolean detach)
Parameters:
detach : Set to 1 if you don't care when this application exists.
Ensures:
$self->isLaunched() == result
Returns:
whether the application was successfully launched.

Launch the application asynchronously. That is, it will not wait until the application has exited.

If $detach is false, then you must periodically call $AppLauncher->check() until it returns true. This is to avoid zombie processes on Unix.

If the launch failed, then you can use $AppLauncher->getError() and $AppLauncher->getErrorCode() to get detailed information about the cause.

You must not call this function more than once. If this function failed, and you want to try launching again, then you must discard this object and create a new one.
