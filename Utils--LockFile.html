Utils::LockFile - Lock files.

This class provides an easy interface for lock files. A lock file is like a mutex or critical section, but it can be used between different processes. It's used to synchronize processes with each other.

Lock files should not be used for file I/O: they should not be read from or written to. On Unix it will work, but on Windows it will fail.

$Utils_LockFile->lock

void $Utils_LockFile->lock()
Requires:
!$self->locked()

Lock this file. If it's already locked, wait until the file is unlocked. The lock is automatically released if this Utils::LockFile instance is destroyed.

Throws IOException if the lockfile cannot be opened or if locking fails.

Throws Utils::LockFile::AlreadyLocked if the precondition is not satisfied.

$Utils_LockFile->tryLock

boolean $Utils_LockFile->tryLock()
Requires:
!$self->locked()

Attempt to lock this file. If it's already locked, returns 0. If it's not already locked, then our lock has succeeded, and 1 is returned. The lock is automatically released if this Utils::LockFile instance is destroyed.

Throws IOException if the lockfile cannot be opened or if locking fails.

Throws Utils::LockFile::AlreadyLocked if the precondition is not satisfied.

$Utils_LockFile->unlock

void $Utils_LockFile->unlock()
Requires:
$self->locked()

Unlock this file.

Throws Utils::LockFile::NotLocked if the precondition is not satisfied.
