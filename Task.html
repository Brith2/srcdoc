Task - Abstract task base class.

This is the abstract base class for all tasks. Please read the AI subsystem and task framework overview for an overview.

Notes on priority constants

The only things you may assume about the values of priority contants are:
  • Each priority constant differ at least a value of 100 from other priority constants. That is, abs(c1 - c2) >= 300 if c1 and c2 are two random priority constants.
  • A higher value means a higher priority.

Details

$Task->activate

void $Task->activate()
Requires:
$self->getStatus() == Task::INACTIVE
Ensures:
$self->getStatus() == Task::RUNNING

Notify a task that it will be activated. Activation happens only once: just before iterate() is called, but only if the task has just been created. This allows the task to perform initialization.

This method will be called by the task manager.

$Task->getError

Hash* $Task->getError()
Requires:
$self->getStatus() == Task::DONE

If the status is Task::DONE, then return information about the error (if the task completed with an error). Otherwise (if the task completed successfully), return undef.

The error information is a reference to a hash, containing two items:

  • code - The error code.
  • message - The error message.

$Task->getMutexes

Array<String>* $Task->getMutexes()
Ensures:
defined(result)

Returns a reference to an array of mutexes for this task. Note that the mutex list may change during a Task's life time. This list must not be modified outside the Task object.

If you override this method, then you must ensure that when the mutex list changes, you trigger a onMutexesChanged event. Otherwise the task manager will not behave correctly.

$Task->getName

String $Task->getName()
Ensures:
$result ne ""

Returns a human-readable name for this task.

$Task->getPriority

int $Task->getPriority()
Get the priority for this task. This priority is guaranteed to never change during a Task's life time.

$Task->getStatus

$Task->getStatus()
Returns the task's status. This is one of Task::RUNNING, Task::INTERRUPTED, Task::STOPPED or Task::DONE.

$Task->interrupt

void $Task->interrupt()
Requires:
$self->getStatus() == Task::RUNNING
Ensures:
$self->getStatus() == Task::INTERRUPTED

Notify a (running) task that it is about to be interrupted. The task may take necessary actions (like updating internal timers) in preparation for interruption. The task must immediately cease all actions, and set the status to Task::INTERRUPTED.

This method should only be called by the task manager.

Task implementors may override this method to implement code for interruption handling.

$Task->iterate

void $Task->iterate()
Requires:
$self->getStatus() == Task::RUNNING

Run one iteration of this task. Task implementors must override this method to implement task code.

$Task->onMutexesChanged

CallbackList $Task->onMutexesChanged()
This event is triggered when the mutex list for this task has changed.

$Task->onStop

CallbackList $Task->onStop()
This event is triggered when the task's status has been set to Task::STOPPED.

$Task->resume

void $Task->resume()
Requires:
$self->getStatus() == Task::INTERRUPTED
Ensures:
$self->getStatus() == Task::RUNNING

Notify an (interrupted) task that it is about to be resumed. The task may take necessary actions in prepration for resuming. The task must set its status to Task::RUNNING.

This method should only be called by the task manager.

Task implementors may override this method to implement code for resume handling.

$Task->setDone

void $Task->setDone()
Requires:
$self->getStatus() == Task::INACTIVE or Task::RUNNING

Indicate that the task has been completed, without error. The status will be set to Task::DONE.

Do not call this method outside $Task->iterate(), or bad things will happen!

$Task->setError

void $Task->setError(code, message)
Parameters:
code : An error code.
message : An error message.
Requires:
$self->getStatus() == Task::INACTIVE or Task::RUNNING

Indicate that the task has been completed, but with an error. The status will be set to Task::DONE, and $Task->getError() will return the error information passed to this method.

Do not call this method outside $Task->iterate(), or bad things will happen!

$Task->setMutexes

void $Task->setMutexes(mutexes...)
Set the currently active mutexes for this task. This will trigger an onMutexesChanged event.

You should only call this method inside the class's iterate() method, or during initialization. Otherwise you may confuse the task manager.

$Task->setStopped

void $Task->setStopped()
Requires:
$self->getStatus() == Task::RUNNING, Task::INACTIVE or Task::INTERRUPTED
Ensures:
$self->getStatus() == Task::STOPPED

Set the task's status to Task::STOPPED and trigger an onStop event. This is useful for tasks that cannot stop immediately when stop() is called: they can mark the task as stopped when appropriate.

$Task->stop

void $Task->stop()
Requires:
$self->getStatus() == Task::RUNNING, Task::INACTIVE or Task::INTERRUPTED

Notify a task that it must completely stop. When the task is actually stopped, the status must be set to Task::STOPPED.

The default behavior is to immediate set the status to Task::STOPPED by calling $Task->setStopped(), thereby triggering an onStop event. Task implementors may override this method to implement custom stop handling. You may choose to stop the task after a period of time, instead of immediately.

This method may be called by anybody, not just the task manager.

Task->new

Task->new(options...)
Ensures:
result->getStatus() == Task::INACTIVE

Create a new Task object. The following options are allowed:
  • name - A name for this task. $Task->getName() will return this name. If not specified, the class's name (excluding the "Task::" prefix) will be used as name.
  • priority - A priority for this task. $Task->getPriority() will return this value. The default priority is Task::NORMAL_PRIORITY
  • mutexes - A reference to an array of mutexes. $Task->getMutexes() will return this value. The default is an empty mutex list.
