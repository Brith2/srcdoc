InventoryList - Inventory model

Derived from: ObjectList

The InventoryList class models a character's inventory or a Kapra storage.

Differences compared to ObjectList

All items in Inventory are of the same class, and are all a subclass of Actor::Item.

$InventoryList->add

int $InventoryList->add(Actor::Item item)
Requires:
defined($item) defined($item->{name}) $self->find($item) == -1
Ensures:
$item->{invIndex} == result

Adds an item to this InventoryList. $item->{invIndex} will automatically be set index in which that item is stored in this list.

This method overloads $ObjectList->add(), and has a stronger precondition. See the documentation for that method for more information about this method.

$InventoryList->getByCondition

Actor::Item $InventoryList->getByCondition(Function condition)
Return the first Actor::Item object for which the function $condition returns true. If nothing is found, undef is returned.

$condition is called with exactly one parameter, namely the item that is currently being checked.

$InventoryList->getByName

Actor::Item $InventoryList->getByName(String name)
Requires:
defined($name)
Ensures:
if defined(result): result->{ID} eq $ID
Returns:
An Actor::Item, or undef if there is no item with that name in this list.

Looks up an Actor::Item object based on the item name. The name lookup is case-insensitive. If there is more than one item with this name, then it is unspecified which exact item will be returned.

See also: $Actor->{ID}

$InventoryList->getByNameID

Actor::Item $InventoryList->getByNameID(Bytes nameID)
Return the first Actor::Item object, whose 'nameID' field is equal to $nameID. If nothing is found, undef is returned.

$InventoryList->getByNameList

Actor::Item $InventoryList->getByNameList(String nameList)
Parameters:
nameList : a string containing a comma-separated list of item names.
Requires:
defined($nameList)
Returns:
The found item, or undef if not found.

Lookup an item by using the specified name list. For example, if $nameList is "Red Potion,White Potion,Jellopy", then this method will look up either Red Potion, White Potion or Jellopy, whichever is found first.

$InventoryList->getByServerIndex

Actor::Item $InventoryList->getByServerIndex(int serverIndex)
Return the first Actor::Item object, whose 'index' field is equal to $serverIndex. If nothing is found, undef is returned.

$InventoryList->remove

boolean $InventoryList->remove(Actor::Item item)
Requires:
defined($item) && defined($item->{name})

Removes an item from this InventoryList.

This method overloads $ObjectList->remove(), and has a stronger precondition. See the documentation for that method for more information about this method.

$InventoryList->removeByName

boolean $InventoryList->removeByName(String name)
Parameters:
name : The name of the item to remove.
Requires:
defined($name)
Returns:
Whether the item with the specified name was in the list.

Removes an item based on the item name. The name lookup is case-insensitive. If there is more than one item with this name, then it is unspecified which exact item (with that name) is removed.

This will trigger an onRemove event before the item is removed.
